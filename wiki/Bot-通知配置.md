# Bot 通知配置

Bot 通知页面地址：

```text
/bot.html
```

这个页面用于：

- 查看 Bot 在线状态
- 添加通知目标
- 启用或禁用目标
- 删除目标

## 启用 Bot 功能

先在配置里启用 Bot：

```json
"Bot": {
  "Enabled": true,
  "Tokens": [
    "bot_token_1",
    "bot_token_2"
  ]
}
```

也可以用环境变量：

- `Bot__Enabled=true`
- `Bot__Tokens__0=...`
- `Bot__Tokens__1=...`

## 支持的目标输入

添加目标时支持两种写法：

- `@username`
- `Chat ID`

例如：

- `@your_group_name`
- `-1001234567890`

## 目标校验规则

当前版本会对“所有已配置 Bot”执行校验。

只有在以下条件全部满足时，目标才会被添加成功：

1. 每个 Bot 都能获取该会话信息
2. 群组、超级群、频道场景下，每个 Bot 都具备可发送消息的权限

这意味着：

- 如果是私聊目标，每个 Bot 都必须和这个用户建立过有效会话
- 如果是群组或频道目标，每个 Bot 都必须已经在群里，并且权限满足要求

## 消息发送策略

- 发送时会在多个 Bot 之间轮询
- 如果当前轮到的 Bot 发送失败，会自动尝试下一个 Bot
- 回调按钮会由收到回调的那个 Bot 负责响应

## 按钮回调

当通知消息带有“屏蔽此人”按钮时，点击后会自动创建一条排除规则，用于屏蔽该发送者。

## 常见问题

### 为什么添加目标时报某个 Bot 校验失败

因为当前逻辑要求所有 Bot 都通过校验，常见原因：

- 某个 Bot 不在目标群或频道里
- 某个 Bot 没有发言权限
- 私聊用户没有和某个 Bot 建立会话

### 为什么建议优先用公开用户名

`@username` 更直观，也更方便排查问题。

但如果目标没有公开用户名，仍然可以使用 `Chat ID`。
