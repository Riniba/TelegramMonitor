# 配置说明

TelegramMonitor 默认从 `appsettings.json` 读取配置，也支持使用环境变量覆盖。

## 最低可用配置

```json
{
  "Urls": "http://*:5005",
  "Telegram": {
    "DefaultApiId": 123456,
    "DefaultApiHash": "your_api_hash",
    "SessionsPath": "session"
  },
  "Auth": {
    "AdminUsername": "admin",
    "AdminPassword": "change-me"
  },
  "DbConnection": {
    "DbType": "Sqlite",
    "ConnectionString": "DataSource=telegrammonitor.db"
  },
  "Bot": {
    "Enabled": false,
    "Tokens": []
  }
}
```

## 配置项说明

### `Urls`

- 默认值：`http://*:5005`
- 用于控制 Web 后台监听地址和端口

### `Telegram`

- `DefaultApiId`
  - 你的 Telegram API ID
  - 账号登录前必须先配置
- `DefaultApiHash`
  - 你的 Telegram API Hash
  - 账号登录前必须先配置
- `SessionsPath`
  - 会话文件保存目录
  - 默认是 `session`

### `Auth`

- `AdminUsername`
  - 管理后台用户名
- `AdminPassword`
  - 管理后台密码
  - 部署前务必修改
- `CookieName`
  - 后台登录 Cookie 名称
- `CookieExpireMinutes`
  - Cookie 有效期，默认 480 分钟
- `SlidingExpiration`
  - 是否启用滑动续期

### `DbConnection`

- `DbType`
  - 默认 `Sqlite`
  - 也支持 SqlSugar 可用的其他数据库类型
- `ConnectionString`
  - 数据库连接字符串
  - SQLite 默认值是 `DataSource=telegrammonitor.db`

### `Bot`

- `Enabled`
  - 是否启用 Bot 通知
- `Tokens`
  - Bot Token 列表
  - 支持多个 Bot

## 环境变量写法

常用环境变量：

- `Urls`
- `Telegram__DefaultApiId`
- `Telegram__DefaultApiHash`
- `Telegram__SessionsPath`
- `Auth__AdminUsername`
- `Auth__AdminPassword`
- `DbConnection__DbType`
- `DbConnection__ConnectionString`
- `Bot__Enabled`
- `Bot__Tokens__0`
- `Bot__Tokens__1`

## 安全建议

- 不要把真实的 `ApiHash`、Bot Token、生产密码提交到 GitHub
- 本地开发建议使用未跟踪的配置文件或环境变量
- 公开部署时优先通过环境变量注入敏感配置
